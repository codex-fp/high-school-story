Product documentation workspace for High School Story.

Use this folder for product-facing documentation such as:
- feature briefs
- lightweight PRDs
- user flows
- requirements and acceptance criteria
- release notes drafts

Keep documents focused on player value, scope, constraints, and measurable outcomes.

Working mode:
- product documentation is created through interview with the game creator
- the assistant asks focused product questions and turns answers into professional documentation
- assumptions must be labeled clearly; unresolved topics go to open questions
