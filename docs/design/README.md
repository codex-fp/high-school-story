# Design Documentation

`docs/design/` is the approved space for topic-specific High School Story design
documentation.

Use this directory when a design topic needs more detail than belongs in the
root `DESIGN.md` overview, such as system-specific mechanics, balancing notes,
progression rules, UX flows, or design records for settled gameplay decisions.

Keep `DESIGN.md` as the top-level product and gameplay contract for player
promise, target experience, MVP scope, and cross-cutting design constraints.
Design files in this directory should link back to `DESIGN.md` when they refine
or depend on those root-level contracts.

Do not put authored scenes or story prose here; keep narrative content in
`docs/narrative/`.
