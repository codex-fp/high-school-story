# Game Design Documentation

This space contains system-level game design documentation for High School Story.

It translates product intent into playable rules, loops, tuning levers, and content-design constraints.

## What This Space Contains

- design interpretation of the product vision
- system rules and player-facing mechanics
- pacing, progression, and balancing frameworks
- identified risks, edge cases, and open design questions

## Relationship To Other Spaces

- `docs/product` defines what the game should deliver and why it matters
- `docs/design` defines how the game should work in play
- `docs/narrative` defines story content, scenes, and written material

## Working Principles

- convert high-level intentions into explicit gameplay rules
- label assumptions clearly and avoid hiding missing decisions
- prefer mechanics that create tradeoffs over mechanics that add complexity only
- document both player-facing rules and behind-the-scenes tuning knobs
- update existing documents when a decision becomes final
